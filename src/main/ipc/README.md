# src/main/ipc — 主 ↔ 渲染 通信层

主进程与渲染进程之间的**唯一通道**。定义并注册 IPC 处理器,保持接口边界清晰(这也是未来把内核拆成独立服务的接缝)。

## 典型消息
- 渲染 → 主:用户输入文本、请求唤出/收起对话框、宠物被拖拽到 (x,y)、打开设置、读写配置。
- 主 → 渲染:流式回复 token、宠物状态切换(idle/thinking/talk/…)、气泡文本 + 音频指令、热键触发。

## 交互
- ↔ [src/renderer](../../renderer/):收发所有跨进程消息。
- → [agent/](../agent/) [config/](../config/) [lines/](../lines/) 等:把渲染层请求路由到对应主进程模块。
- → [src/shared](../../shared/):复用消息类型与通道常量,保证两端类型一致。
